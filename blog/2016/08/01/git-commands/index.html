Git Merge & Git Rebase

觀念：只要branch 調整完畢，應先將master merge 至branch，測試無誤後，在merge 回master

merge 分支的功能加master，同時會產生新的HEAD

合併分支到master

rebase 是基於某一分支的內容合併，又稱為分支衍合，一樣是合併的動作，但是會銜接著mater 最新的狀態

Merge v.s. Rebase
